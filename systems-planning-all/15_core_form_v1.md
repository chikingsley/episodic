# Core Form v1

## Purpose

This note defines the base playable form more tightly than the broader product notes.

The goal is to stop the project from expanding sideways every time a new idea appears.

## One-Sentence Definition

Dark Mallard should play as a **VN-style conversation puzzle thriller** where **Pimsleur-style rehearsal feeds into authored, stateful dialogue encounters**.

## What `VN` Means

`VN` means `visual novel`.

In practice, that means:

- a mostly static or semi-static scene
- one or more characters presented directly on screen
- dialogue and response UI as the main interaction surface
- strong writing and state changes carrying the tension more than locomotion or combat

This does **not** mean the game has to be a pure visual novel.

It means the presentation grammar is closer to:

- portrait or half-body character framing
- dialogue boxes
- meters, casefiles, prompts, or overlays
- authored scene transitions

than to:

- open-world traversal
- RPG exploration
- freeform AI chat

## Core Form

The project has two linked layers:

### 1. Rehearsal Layer

This is the Pimsleur-style layer.

It is for:

- hear
- repeat
- distinguish
- build automaticity
- rehearse small constructions under mild pressure

It should feel compact, guided, and efficient.

### 2. Encounter Layer

This is the game layer.

It is for:

- deploy what was rehearsed
- choose what move to make next
- deal with uncertainty
- manage social state
- absorb consequences

It should feel tense, authored, and stateful.

## What The Puzzle Actually Is

The puzzle is **not**:

- solving a fictional language
- guessing arbitrary words
- chatting freely until something works

The puzzle is:

**Given limited language, limited knowledge, and scene pressure, what is the best move right now?**

That means the player is solving a combination of:

- intent puzzle
- sequencing puzzle
- state-management puzzle
- information puzzle

### Intent Puzzle

What kind of move should this line perform?

Examples:

- ask
- confirm
- deny
- calm
- stall
- redirect
- end interaction

### Sequencing Puzzle

What should be said first, second, and not yet?

Examples:

- calm first, ask second
- verify identity before giving information
- gather location before escalating

### State-Management Puzzle

What will this move do to the person and the scene?

Examples:

- increase trust
- trigger suspicion
- waste time
- unlock new prompts
- close off a route

### Information Puzzle

What do I know, what do I still need, and who should I pressure or reassure to get it?

That is where the game part lives.

## Why This Is Still A Game At Beginner L2

Beginner L2 does not give the player expressive freedom.

So the game cannot depend on:

- saying anything
- roleplaying freely
- improvising like a fluent speaker

Instead, the game depends on:

- constrained verbs
- meaningful scene state
- uncertainty
- consequence

That is closer to `Papers, Please` than to an RPG town.

The player is not powerful because they can say everything.

The player is powerful because they can make a few meaningful moves at the right time.

## The Basic Loop

1. rehearse a small set of constructions
2. enter a bounded dialogue encounter
3. choose which speech move to make
4. speak
5. change the state of the scene
6. unlock, lose, or reshape the next options
7. finish with consequence

## The Smallest Unit Of Play

The smallest satisfying unit is not a whole mission.

It is:

- one scene
- one objective
- one emotional or tactical pressure
- one limited speech toolkit

Example:

- objective: get the location
- pressure: the other person is upset or suspicious
- toolkit: reassure, ask where, ask who, ask when, repeat, wait

That is already enough for play if the state changes matter.

## What The Player Is Actually Choosing

The player is not mainly choosing between giant branching story options.

The player is choosing between small tactical speech moves such as:

- push now
- soften now
- verify now
- ask for repetition
- switch target
- spend help
- leave

That should be the center of the design.

## What Is Not The Core Form

These may still exist, but they are not the foundation:

- open-world living simulation
- freeform AI conversation
- large branching romance web
- giant RPG progression tree
- decipherment puzzle language
- minigame wrappers around vocab quizzes

## Where Plot Fits

Plot matters, but it sits on top of form.

The form comes first:

- what the player sees
- what the player can do
- how state changes
- where tension comes from

Then plot gives that form a reason to matter.

So the correct order is:

1. lock the core form
2. lock the scene machine
3. lock the recurring pressure types
4. then build plot, factions, and episodes on top

## Practical Design Constraint

If a scene cannot be described in terms of:

- objective
- available speech moves
- state variables
- failure pressure
- consequence

then it is still too vague to build.

## Working Tagline

**A VN-style conversation puzzle thriller where rehearsal becomes deployment.**
